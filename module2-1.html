2.1 PRD(제품 요구사항 문서) 작성

PRD란? Product Requirements Document의 약자로, 제품이나 기능의 목적, 요구사항, 성공 지표를 정의하는 핵심 문서입니다.

PRD 작성 워크플로우

graph TD A[요구사항 수집] --> B[AI에게 초안 요청] B --> C[AI 초안 생성] C --> D{PM 검토} D -->|수정 필요| E[피드백 반영 요청] E --> C D -->|승인| F[이해관계자 리뷰] F --> G{최종 승인} G -->|수정 필요| E G -->|승인| H[PRD 확정 및 공유]

AI로 PRD 작성하기 — 단계별 가이드

  1. Step 1: 배경과 목적 정의

    기능의 배경, 해결할 문제, 대상 사용자를 AI에게 전달합니다.

  2. Step 2: 요구사항 도출

    기능적/비기능적 요구사항을 AI와 함께 브레인스토밍합니다.

  3. Step 3: 성공 지표 설정

    KPI와 측정 방법을 구체적으로 정의합니다.

  4. Step 4: 리스크 분석

    예상 리스크와 대응 방안을 AI에게 요청합니다.

프롬프트 예시

PRD 작성 프롬프트
당신은 B2B SaaS 제품의 시니어 PM입니다.

다음 기능의 PRD를 작성해주세요:

## 배경
- 제품: 프로젝트 관리 도구
- 기능: 팀 대시보드 실시간 현황판
- 문제: 팀장이 팀원들의 작업 현황을 파악하기 어려움
- 대상: 5~20명 규모 팀의 팀장

## PRD 구조
1. 기능 개요 (목적, 배경, 대상 사용자)
2. 사용자 스토리 (3가지 이상)
3. 기능 요구사항 (테이블: ID, 요구사항, 우선순위, 상태)
4. 비기능 요구사항 (성능, 보안, 접근성)
5. 성공 지표 (KPI 3가지, 측정 방법 포함)
6. 타임라인 (4주 기준)
7. 리스크 및 대응 방안

마크다운 형식으로 작성해주세요.

실습 과제

여러분의 실제 프로젝트에서 하나의 기능을 선택하여 위 프롬프트 템플릿을 수정하고, AI로 PRD를 생성해보세요.

2.2 기획서/제안서 자동 생성

기획서와 제안서는 이해관계자를 설득하기 위한 문서입니다. AI를 활용하면 초안 작성 시간을 크게 단축할 수 있습니다.

기획서 구조 템플릿

섹션 내용 AI 활용 방법
1. 요약 핵심 내용을 1페이지로 정리 전체 기획 내용을 요약 요청
2. 문제 정의 해결할 문제와 현재 상황 문제 분석 및 근거 데이터 요청
3. 솔루션 제안하는 해결 방안 여러 대안 비교표 생성
4. 실행 계획 일정, 리소스, 마일스톤 WBS 및 간트차트 구조 요청
5. 기대 효과 정량적/정성적 효과 ROI 계산 프레임워크 요청
기획서 작성 프롬프트
다음 프로젝트의 기획서를 작성해주세요.

[프로젝트 개요]
- 프로젝트명: 고객 피드백 자동 분류 시스템
- 목표: CS팀의 피드백 분류 시간을 50% 단축
- 예산: 3,000만원
- 기간: 3개월

포함할 항목:
1. Executive Summary (1페이지)
2. 현재 문제점 분석 (데이터 기반)
3. 제안 솔루션 3가지 비교표
4. 선택한 솔루션의 상세 실행 계획
5. 예상 ROI 및 기대 효과
6. 리스크 매트릭스 (영향도 x 발생확률)

2.3 요구사항 정의서 작성

요구사항 정의서는 개발팀에게 "무엇을 만들어야 하는지" 전달하는 문서입니다.

요구사항 유형

유형 설명 예시
기능 요구사항 (FR) 시스템이 수행해야 할 기능 "사용자는 이메일로 로그인할 수 있다"
비기능 요구사항 (NFR) 성능, 보안, 사용성 등 "페이지 로딩은 3초 이내"
비즈니스 규칙 (BR) 업무 로직과 제약 조건 "무료 플랜은 프로젝트 3개까지"
요구사항 도출 프롬프트
다음 기능의 요구사항 정의서를 작성해주세요.

기능: 팀 멤버 초대 및 권한 관리
대상: SaaS 프로젝트 관리 도구

다음 형식의 테이블로 정리해주세요:

| ID | 유형 | 요구사항 | 우선순위 | 수용 기준 |
|----|------|---------|---------|----------|

- 기능 요구사항 최소 10개
- 비기능 요구사항 최소 5개
- 각 요구사항에 테스트 가능한 수용 기준 포함
- 우선순위: Must/Should/Could/Won't (MoSCoW)

2.4 회의록/보고서 자동 정리

AI를 활용하면 회의록 작성과 보고서 정리 시간을 크게 줄일 수 있습니다.

회의록 자동 정리 파이프라인

graph LR A[회의 녹음/메모] --> B[음성 전사
Clova Note, Otter] B --> C[AI 구조화
Claude, ChatGPT] C --> D[액션 아이템 추출] D --> E{PM/PL 검토} E -->|보완 필요| C E -->|확인 완료| F[팀 공유
Slack, Notion]

회의록 정리 워크플로우

  1. 회의 내용 입력

    회의 중 메모한 내용이나 녹취록을 AI에 입력합니다.

  2. 구조화 요청

    AI에게 정해진 형식으로 정리해달라고 요청합니다.

  3. 액션 아이템 추출

    담당자, 기한이 포함된 액션 아이템을 별도 추출합니다.

회의록 정리 프롬프트
다음 회의 내용을 정리해주세요.

[회의 메모]
(여기에 회의 메모를 붙여넣기)

정리 형식:
1. 회의 개요 (일시, 참석자, 목적)
2. 주요 논의 사항 (bullet point)
3. 결정 사항 (번호 매기기)
4. 액션 아이템 테이블:
   | 번호 | 항목 | 담당자 | 기한 | 상태 |
5. 다음 회의 일정 및 안건

음성 기반 회의록 자동 생성 워크플로우

2025+ 트렌드: 텍스트 메모 대신 회의 녹음을 AI가 자동으로 텍스트 변환하고 구조화하는 워크플로우가 표준이 되고 있습니다.
도구 특징 활용 방법
Clova Note 한국어 특화, 화자 분리, 무료 플랜 제공 한국어 회의 녹음 → 텍스트 변환 → AI로 구조화
Otter.ai 영어 특화, 실시간 전사, 요약 자동 생성 영어 회의/글로벌 콜 → 실시간 녹취 → 요약 추출
Whisper (OpenAI) 오픈소스, 다국어 지원, 높은 정확도 녹음 파일 → Whisper 변환 → Claude/ChatGPT로 분석
MS Teams Copilot Teams 내장, 자동 요약 및 액션아이템 추출 Teams 회의 → 자동 녹취 → Copilot 요약 → 내보내기
  1. 회의 녹음

    참석자 동의 하에 회의를 녹음합니다. 스마트폰 녹음 앱이나 화상회의 녹화 기능을 활용합니다.

  2. 음성 → 텍스트 변환 (STT)

    Clova Note, Whisper 등으로 녹음 파일을 텍스트로 변환합니다. 화자 분리가 되면 더욱 유용합니다.

  3. AI로 구조화 및 요약

    변환된 텍스트를 Claude나 ChatGPT에 입력하여 회의록 형식으로 정리합니다.

  4. 액션 아이템 추출 및 공유

    AI가 추출한 결정 사항과 액션 아이템을 Jira, Notion 등 협업 도구에 등록합니다.

음성 녹취록 → 회의록 변환 프롬프트
다음은 회의 녹음의 텍스트 변환 결과입니다. 공식 회의록으로 정리해주세요.

[녹취록]
(STT 도구로 변환된 텍스트를 붙여넣기)

정리 요청:
1. 회의 개요 (일시, 참석자, 안건)
2. 안건별 논의 요약 (핵심만 간결하게)
3. 결정 사항 (번호 매기기)
4. 액션 아이템 테이블:
   | 번호 | 항목 | 담당자 | 기한 | 우선순위 |
5. 미결 사항 및 후속 논의 필요 항목
6. 다음 회의 일정
