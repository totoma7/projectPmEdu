7. AI 프로젝트 실전 케이스 스터디

앞서 배운 AI 활용 기법들을 엔드투엔드로 연결하여, 실제 프로젝트에서 PM과 PL이 AI를 어떻게 활용하는지 통합 시나리오로 학습합니다.

7.1 케이스 1: 신규 SaaS 제품 기획~MVP 출시 (PM 관점)

시나리오: 중소기업 대상 "AI 기반 고객 피드백 분석 SaaS"를 기획하고, 바이브 코딩으로 MVP를 제작하여 시장에 출시하는 전체 프로세스입니다.

SaaS MVP 출시 전체 프로세스

graph TD A[시장 기회 발굴
Module 4: AI 리서치] --> B[사용자 리서치
페르소나/인터뷰] B --> C[PRD 작성
Module 2: AI 문서] C --> D[MVP 개발

전체 프로세스 오버뷰

단계 활용 모듈 AI 활용 내용 산출물
1. 시장 조사 Module 4 AI 검색 도구로 시장 규모 조사, 경쟁사 분석 시장 분석 보고서, 경쟁사 Feature Matrix
2. 사용자 리서치 Module 4 AI로 인터뷰 질문지 설계, 페르소나 생성 페르소나 3명, 인터뷰 분석 보고서
3. 기획 Module 2 AI로 PRD 작성, 기획서 생성, 요구사항 정의 PRD, 기능 요구사항 정의서
4. 설계 Module 3 AI로 시스템 아키텍처, ER 다이어그램, 사용자 여정 맵 아키텍처 다이어그램, 화면 흐름도
5. MVP 개발 Module 5 Claude Code로 바이브 코딩, 반복 개선 동작하는 MVP 웹 애플리케이션
6. 검증 및 출시 Module 4, 2 A/B 테스트 설계, 릴리스 노트 작성 테스트 결과 보고서, 출시 문서

Step 1: AI로 시장 기회 발굴

시장 기회 분석 프롬프트
한국 중소기업(50~200명) 대상 고객 피드백 분석 SaaS 시장을 조사해주세요.

분석 항목:
1. 시장 규모 (TAM/SAM/SOM 추정, 출처 포함)
2. 기존 솔루션과 한계 (Zendesk, 채널톡, 직접 구축)
3. 중소기업이 겪는 핵심 Pain Point 5가지
4. 차별화 기회 (AI 기반 자동 분류, 감성 분석, 트렌드 예측)
5. Go-to-Market 전략 제안

결론을 "투자할 가치가 있는가?"로 정리해주세요.

Step 2: PRD 작성 및 검토

MVP PRD 작성 프롬프트
다음 시장 조사 결과를 바탕으로 MVP PRD를 작성해주세요.

[시장 조사 요약]
(Step 1 결과를 붙여넣기)

MVP 범위:
- Phase 1(출시): 피드백 수집 + AI 자동 분류 + 기본 대시보드
- Phase 2(1개월 후): 감성 분석 + 트렌드 리포트
- Phase 3(3개월 후): 액션 아이템 자동 생성 + Slack 연동

PRD 구조:
1. 제품 비전 및 목표
2. 타겟 사용자 및 페르소나
3. MVP 기능 요구사항 (MoSCoW 우선순위)
4. 성공 지표 (DAU, 피드백 분류 정확도, NPS)
5. 기술 제약사항 및 의존성
6. 4주 개발 타임라인

Step 3: 바이브 코딩으로 MVP 제작

Claude Code MVP 개발 프롬프트
고객 피드백 분석 SaaS의 MVP를 만들어주세요.

## 핵심 기능
1. 피드백 입력 폼 (텍스트, 별점)
2. AI 자동 분류 (카테고리: 기능요청/버그/칭찬/불만/기타)
3. 대시보드 (카테고리별 비율, 최근 피드백 목록, 감성 트렌드)
4. 간단한 관리자 페이지

## 기술 스택
- Frontend: React + Tailwind CSS
- Backend: Node.js + Express
- Database: SQLite (MVP용)
- AI: OpenAI API (분류용)

## 요구사항
- 반응형 디자인 (모바일 대응)
- 다크모드 지원
- 데모 데이터 50건 포함

Step 4: 검증 및 출시 준비

출시 체크리스트 프롬프트
MVP 출시를 위한 체크리스트와 릴리스 노트를 작성해주세요.

## 출시 체크리스트
- 기능 테스트 완료 여부
- 보안 점검 (API 키 노출, SQL 인젝션 등)
- 성능 테스트 (동시 사용자 50명 기준)
- 랜딩 페이지 및 온보딩 가이드
- 피드백 수집 채널 준비

## 릴리스 노트
- 대상: 초기 베타 사용자 20개 기업
- 어투: 친근하지만 전문적
- 포함: 핵심 기능 소개, 사용 가이드 링크, 피드백 요청

종합 실습

위 4단계를 직접 수행하여, 여러분만의 SaaS 아이디어를 시장 조사 → PRD → MVP까지 만들어보세요. 전체 과정에서 AI를 어떻게 활용했는지, 사람이 직접 한 부분은 무엇인지를 기록하세요.

7.2 케이스 2: 레거시 시스템 마이그레이션 프로젝트 관리 (PL 관점)

시나리오: 10년된 온프레미스 ERP 시스템을 클라우드 SaaS로 마이그레이션하는 6개월 프로젝트를 AI를 활용하여 관리합니다.

레거시 마이그레이션 타임라인

gantt title ERP 클라우드 마이그레이션 (6개월) dateFormat YYYY-MM-DD axisFormat %m월 section 분석 현황 분석 및 요구사항 수집 :a1, 2026-01-01, 4w AI 활용 리스크 분석 :a2, after a1, 2w section 설계 아키텍처 설계 :b1, after a2, 3w 데이터 마이그레이션 계획 :b2, after a2, 3w section 개발 코어 모듈 마이그레이션 :c1, after b1, 6w API 통합 개발 :c2, after b1, 4w section 테스트 통합 테스트 :d1, after c1, 3w 사용자 수용 테스트(UAT) :d2, after d1, 2w section 전환 데이터 마이그레이션 실행 :e1, after d2, 1w Go-Live 및 모니터링 :e2, after e1, 2w

프로젝트 개요

항목 내용
프로젝트명 ERP 클라우드 마이그레이션
기간 6개월 (24주)
팀 규모 개발 5명, QA 2명, DBA 1명, 인프라 1명, PM 1명
핵심 리스크 데이터 무결성, 업무 중단 최소화, 사용자 저항

PL의 AI 활용 타임라인

  1. Phase 1 (1~4주): 분석 및 계획

    AI로 현행 시스템 분석 보고서 작성, WBS 3단계 분해, 리스크 관리대장 초안 생성, 이해관계자 분석 및 커뮤니케이션 계획 수립

  2. Phase 2 (5~12주): 설계 및 개발

    AI로 주간 상태 보고서 자동 생성, 기술 이슈 분석 지원, 일정 영향 분석, 데일리 스탠드업 메모 자동 정리

  3. Phase 3 (13~20주): 테스트 및 데이터 마이그레이션

    AI로 테스트 시나리오 생성, 데이터 검증 체크리스트 작성, 롤백 계획 문서화, QA 결과 분석 및 리포트

  4. Phase 4 (21~24주): 전환 및 안정화

    AI로 전환 매뉴얼 작성, 사용자 교육 자료 생성, 장애 대응 매뉴얼, 프로젝트 종료 보고서

마이그레이션 WBS 생성 프롬프트
온프레미스 ERP를 클라우드 SaaS로 마이그레이션하는 프로젝트의 WBS를 작성해주세요.

## 프로젝트 정보
- 기간: 24주
- 팀: 개발 5명, QA 2명, DBA 1명, 인프라 1명
- 현행 시스템: Java/Oracle 기반 ERP (모듈: 회계, 인사, 재고, 영업)
- 목표 시스템: 클라우드 네이티브 SaaS (AWS)

## WBS 요구사항
1. 4개 Phase로 구분
2. 각 Phase는 5~8개 Task
3. 각 Task는 2~4개 Sub-task
4. 마일스톤 6개 지정
5. 의존관계 및 크리티컬 패스 표시
6. 테이블 형식:
   | WBS ID | Phase | Task | Sub-task | 담당 | 소요(주) | 선행 | 마일스톤 |
마이그레이션 리스크 분석 프롬프트
ERP 클라우드 마이그레이션 프로젝트의 리스크를 분석해주세요.

## 프로젝트 특성
- 10년된 레거시 시스템 (문서화 부족)
- 일일 트랜잭션 약 10만건
- 24/7 운영 시스템 (다운타임 최소화 필수)
- 사용자 300명 (변화 저항 예상)

## 분석 요청
1. 기술 리스크 5개
2. 일정 리스크 3개
3. 조직/인적 리스크 3개
4. 데이터 리스크 3개

각 리스크에 대해:
| 리스크 | 발생확률 | 영향도 | 등급 | 대응전략 | 조기경보 신호 |

종합 실습

가상의 마이그레이션 프로젝트를 설정하고, AI를 활용하여 WBS, 리스크 관리대장, 커뮤니케이션 계획을 생성해보세요. PL로서 AI 결과물을 어떻게 검토하고 수정했는지 기록하세요.

7.3 케이스 3: AI 도구 도입 제안서 작성 (PM+PL 협업)

시나리오: 조직에 AI 도구(Claude, GitHub Copilot 등)를 공식 도입하기 위한 제안서를 작성하고, 파일럿 프로그램을 계획합니다. PM이 전략/ROI를, PL이 실행 계획/리스크를 담당합니다.

YouTube 검색 전략
YouTube에서 AI 도구 활용법을 효과적으로 검색하는 방법:

1. 검색 키워드 조합
   - "[도구명] tutorial 2025" → 최신 버전 튜토리얼
   - "[업무] AI workflow" → 예: "proposal writing AI workflow"
   - "AI tools for [직무]" → 예: "AI tools for project managers"
   - "[도구A] + [도구B] workflow" → 도구 조합 검색

2. 추천 검색 키워드
   - 한국어: "AI 업무 자동화", "AI 툴 추천 2025", "ChatGPT 업무 활용"
   - 영어: "AI productivity workflow", "best AI tools for business"
   - 실전형: "AI로 제안서 만들기", "AI 마케팅 자동화 실전"

3. 필터링 팁
   - 업로드 날짜: 최근 1개월 (AI 도구는 빠르게 변화)
   - 정렬: 조회수순 (검증된 콘텐츠 우선)
   - 영상 길이: 10~20분 (실습 포함 콘텐츠)

AI 툴 조합 워크플로우

  1. Step 1: 리서치 (Perplexity / Claude)

    주제에 대한 배경 조사, 시장 데이터 수집, 경쟁사 분석. 출처가 포함된 정보를 확보합니다.

  2. Step 2: 초안 작성 (ChatGPT / Claude)

    수집된 자료를 바탕으로 구조화된 텍스트 초안을 생성합니다. 목적에 맞는 어투와 분량을 지정합니다.

  3. Step 3: 시각화 (Napkin.ai / Whimsical)

    핵심 개념과 프로세스를 다이어그램, 인포그래픽, 차트로 변환합니다.

  4. Step 4: 프레젠테이션 (Gamma / Beautiful.ai)

    텍스트와 시각 자료를 발표용 슬라이드로 자동 구성합니다.

  5. Step 5: 보완 (이미지/영상/음성)

    필요시 Midjourney로 삽화, ElevenLabs로 나레이션, HeyGen으로 AI 발표 영상을 추가합니다.

주의: AI 도구들은 빠르게 변화합니다. 새 도구를 도입할 때는 반드시 (1) 무료 체험으로 먼저 테스트, (2) 보안/개인정보 정책 확인, (3) 기존 워크플로우와의 호환성을 점검하세요. 유료 결제 전 최소 2주간 실제 업무에 적용해보는 것을 권장합니다.

    NotebookLM이 전체 소스를 분석한 후, 채팅 인터페이스로 질문합니다. "시장 규모 데이터를 정리해줘", "성공 사례의 공통점은?" 등 구체적 질문으로 인사이트를 추출합니다.

  3. Step 3: 산출물 생성

    브리핑 문서, FAQ, 스터디 가이드, 타임라인 등 원하는 형식으로 정리를 요청합니다. 오디오 오버뷰 기능으로 팟캐스트 형태의 요약도 생성할 수 있습니다.

  4. Step 4: 다른 AI 도구와 연결

    NotebookLM에서 추출한 인사이트를 Claude에 전달하여 제안서 본문 작성, Gamma에 전달하여 PPT 제작, Napkin.ai에 전달하여 인포그래픽 생성 등 후속 작업에 활용합니다.
