학습 로드맵 — 이 사이트를 어떻게 사용하나요?

처음 방문하셨다면 이 10분을 투자하세요. 역할과 목표에 따라 최단 경로를 안내합니다.

1. 이 사이트의 구성

  • 10개 본 모듈: AI 기초 → 문서 작성 → 도식화 → 자료 조사 → 바이브 코딩 → PM/PL 역할 → 케이스 스터디 → 비용/ROI → 보안·거버넌스 → 2026 트렌드
  • 8개 부록(resource_*): 특정 주제 심화 — MCP 개발, 한국 규제, 프롬프트 카탈로그, 에이전트 패턴, LLM 평가, 1M 컨텍스트, 온디바이스 LLM, 셀프 체크리스트
  • 1개 용어 사전(확장판): 150+ 용어
  • CHANGELOG / README: 최신 갱신 이력

2. 역할별 최적 경로

👨‍💻 개발자 — 바이브 코딩으로 시작해 안정성·거버넌스로

  1. 5장 바이브 코딩 실습 — 도구를 먼저 익히세요 (2~3시간)
  2. 부록 A. MCP 서버 개발 — 사내 시스템 연결 기술 (1시간)
  3. 부록 C. 프롬프트 패턴 카탈로그 — 프롬프트 품질 (30분)
  4. 1장 AI 기초 — 모델·RAG·에이전트 원리 (1시간)
  5. 부록 D. 에이전트 패턴 심화 + 부록 E. LLM 평가 (1.5시간)
  6. 9장 AI 코딩 도구 & 보안 + 부록 B. 한국 규제 (1시간)
  7. 10장 2026 트렌드 + 부록 F. 1M 컨텍스트 + 부록 G. 로컬 LLM (1.5시간)

총 9~10시간 투자로 프로덕션 수준 AI 개발자 역량 확보.

📋 PM · 기획자 — 문서·리서치 자동화로 빠른 ROI

  1. 1장 AI 기초 — 용어와 한계 이해 (1시간)
  2. 2장 AI 활용 문서 작성 + 부록 C. 프롬프트 패턴 (2시간)
  3. 4장 자료 조사 — 시장·경쟁·페르소나 (1.5시간)
  4. 6장 PM vs PL 역할 — 협업 구도 (1시간)
  5. 7장 케이스 스터디 — 국내외 실제 사례 (1시간)
  6. 부록 B. 한국 규제 — 출시 전 필수 (30분)
  7. 8장 비용/ROI — 상신용 자료 (1시간)

총 8시간. 다음날부터 리서치·문서 시간 30~50% 단축 체감.

🎯 PL · 팀 리더 — 거버넌스와 ROI가 먼저

  1. 6장 PM vs PL 역할 이해 — 리더십·팀 리딩 (1.5시간)
  2. 8장 비용/ROI 관리 — 예산과 설득 (1.5시간)
  3. 9장 AI 코딩 도구 & 보안 — 거버넌스 프레임워크 (1.5시간)
  4. 부록 B. 한국 규제 — 법적 리스크 체크리스트 (45분)
  5. 7장 케이스 스터디 — 벤치마크와 의사결정 (1시간)
  6. 10.5~10.6 AI 네이티브 협업 + 하네스 — 팀 운영의 미래 (45분)

총 7시간. 도입 전략 상신 자료 작성 능력 확보.

4. 학습 팁

  • 읽지만 말고 직접 타이핑: 프롬프트 예시는 그대로 복사해서 실제 AI에 입력해 보세요. 결과를 비교해야 감이 옵니다.
  • 한 모듈 = 한 실습: 모듈 하나를 끝낼 때마다 자기 업무에 1가지라도 적용. 미루면 증발합니다.
  • 주 1회 리프레시: 모델·가격·도구는 매달 바뀝니다. CHANGELOG로 따라잡으세요.
  • 혼자보다 팀: 같은 모듈을 2~3명이 함께 읽고 실습 공유 → 학습 속도 3배.
  • 막히면 Ctrl+K: 사이트 내 전체 검색으로 용어·모듈을 바로 찾을 수 있습니다.
