부록 G. 온디바이스·로컬 LLM 실전 가이드

클라우드 LLM이 쓰기 어려운 상황(금융·의료 규제, 기밀 정보, 대량 저비용 처리)에서 로컬에서 LLM을 돌리는 법. 2026-04 기준 실용 수준에 도달한 오픈 모델과 실행 도구를 정리했습니다.

G.1 언제 로컬 LLM을 쓰나

시나리오이유
기밀 문서 요약외부 전송 금지 — 국외 이전 동의 회피
대량 분류·태깅건당 0.001¢ 이하 가능 — 클라우드 대비 100배 저렴
오프라인/엣지공장·매장·선박 — 인터넷 부재
반복 자동화토큰 단가 없음 — 하드웨어 감가만 고려
데모·개발API 키 없이 팀 전체가 무료 시험

안 맞는 경우

  • 최고 수준 추론 필요(수학·고난도 코드) — Claude Opus 4.7·GPT-5가 여전히 유리
  • 아주 적은 호출량 — 하드웨어 고정비가 큰
  • 긴 컨텍스트(>256k) — 로컬 모델은 대부분 128k 이하

G.4 실행 도구 선택

  • Ollama — CLI 한 줄(ollama run llama4). 서버 모드도 내장. 가장 빠른 시작.
  • LM Studio — GUI. 비개발자도 클릭만으로. 모델 검색·교체 쉬움.
  • llama.cpp — 저레벨 엔진. Ollama·LM Studio 내부. 임베디드·커스텀 빌드용.
  • vLLM — 프로덕션 서버. 배치·streaming 최적화, OpenAI 호환 API.
  • Text Generation Inference (TGI) — Hugging Face의 서버. 엔터프라이즈 기능.
  • MLX (Apple) — Apple Silicon 네이티브. M시리즈에서 최고 효율.

G.5 5분 퀵스타트 (Ollama)

# 설치 (mac/linux)
curl -fsSL https://ollama.com/install.sh | sh
# Windows는 ollama.com/download 에서 설치 파일

# 모델 당기기 + 실행
ollama pull qwen3:14b
ollama run qwen3:14b "한국어로 자기소개해줘"

# OpenAI 호환 API가 자동 기동됨 (localhost:11434/v1)
curl http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen3:14b",
    "messages": [{"role":"user","content":"한국 PM 업계 동향 3가지"}]
  }'

Python/TypeScript SDK는 base_url=http://localhost:11434/v1만 바꾸면 OpenAI 코드를 그대로 사용.

G.6 Claude Code·Cursor에 로컬 LLM 연결

최신 Claude Code/Cursor는 OpenAI 호환 엔드포인트를 지원합니다. 시크릿 없이 개인 실험 가능:

// ~/.cursor/config.json 예
{
  "openai": {
    "baseURL": "http://localhost:11434/v1",
    "apiKey": "ollama",
    "model": "qwen3:14b"
  }
}

주의: 로컬 모델은 툴 호출(Function calling) 품질이 클라우드 대비 낮음. 에이전트형 작업은 70B 이상 권장.

G.7 한국어 품질 팁

  • 한국어 비중 높은 모델: EXAONE 4, Qwen 3, HyperCLOVA X Seed
  • 벤치마크 확인: Open Ko-LLM Leaderboard
  • 영어 모델을 강제로 한국어로 쓰면 어색함 발생 — 시스템 프롬프트에 "모든 응답은 자연스러운 한국어로. 어색하면 재작성" 명시
  • 파인튜닝: QLoRA로 노트북에서도 가능. 사내 문서 1,000건으로 도메인 특화 3~6시간

G.8 보안·운영 체크리스트

항목확인
네트워크Ollama 기본 포트 11434가 외부에 노출되지 않도록(사내망 바인딩)
모델 출처Hugging Face 검증된 계정만. 임의 gguf 파일은 악성 코드 가능성(Pickle 공격)
라이선스Llama 4는 월 7억 MAU 미만까지 무료. Qwen 3는 Apache 2.0. 상업 사용 전 확인
로그개인정보·비밀을 모델에 넣었다면 로그/캐시에도 남을 수 있음 — 정기 삭제
갱신모델은 빠르게 진화. 분기 1회 벤치 재평가

G.9 하이브리드 아키텍처 추천

[민감/대량 작업]  → 로컬 Qwen 3 14B (Ollama)
[고난도 추론]     → Claude Opus 4.7 (클라우드)
[라우터]          → 질문 난이도 분류 후 위 둘 중 선택
[가드]            → 개인정보 검출 시 무조건 로컬로 강제

이 구조만으로 비용 40~60% 절감하면서 난이도 높은 질문 품질 유지가 가능합니다.
