4. AI 자료 조사 활용

AI를 활용하여 시장 조사, 경쟁사 분석, 페르소나 생성 등 PM/PL의 리서치 업무를 효율화하는 방법을 학습합니다.

4.1 시장 조사 및 트렌드 분석

핵심: AI는 시장 조사의 "초안"을 빠르게 만들어주지만, 수치와 출처는 반드시 별도 검증해야 합니다.

AI 활용 시장 조사 프레임워크

  1. 시장 규모 파악 (TAM/SAM/SOM)

    전체 시장 → 접근 가능 시장 → 실제 획득 가능 시장을 단계별로 분석합니다.

  2. 트렌드 분석

    업계 동향, 기술 트렌드, 소비자 행동 변화를 파악합니다.

  3. 기회 영역 도출

    시장 분석 결과에서 자사가 진입할 수 있는 기회를 식별합니다.

시장 조사 프롬프트
당신은 시장 분석 전문가입니다.

다음 시장에 대한 조사 보고서를 작성해주세요:

시장: 한국 프로젝트 관리 SaaS 시장

포함할 내용:
1. 시장 규모 (TAM/SAM/SOM 추정)
   - 글로벌 및 국내 시장 규모
   - 연평균 성장률 (CAGR)
2. 주요 트렌드 5가지
   - 각 트렌드별 근거와 영향도
3. 주요 플레이어 및 시장 점유율
4. 진입 기회 및 위협 요인
5. 향후 3년 전망

각 수치에는 출처나 추정 근거를 명시해주세요.
테이블과 bullet point를 활용하여 가독성을 높여주세요.

[주의] AI가 생성한 수치는 추정치이므로,
실제 보고서에 사용하기 전에 반드시 검증하세요.

4.2 경쟁사 분석 프레임워크

경쟁사 분석은 자사 제품의 포지셔닝과 차별화 전략을 수립하는 데 핵심적입니다.

분석 프레임워크

프레임워크 용도 AI 활용 방법
Feature Matrix 기능별 비교표 경쟁사 웹사이트 정보 기반 비교표 생성
SWOT 분석 강점/약점/기회/위협 각 경쟁사별 SWOT 자동 생성
Porter's 5 Forces 산업 경쟁력 분석 5가지 힘에 대한 분석 보고서
가격 비교 가격 포지셔닝 경쟁사 가격표 비교 테이블
포지셔닝 맵 시장 내 위치 시각화 2축 기반 포지셔닝 다이어그램
경쟁사 분석 프롬프트
다음 제품의 경쟁사 분석을 수행해주세요.

자사 제품: 중소기업용 프로젝트 관리 도구
경쟁사: Jira, Asana, Monday.com, Notion, ClickUp

분석 항목:
1. Feature Matrix (핵심 기능 10개 기준 비교표)
   - O: 지원, △: 부분 지원, X: 미지원
2. 가격 비교 테이블 (무료/스타터/프로/엔터프라이즈)
3. 각 경쟁사별 SWOT 분석 (간략)
4. 자사 차별화 포인트 3가지 제안
5. 포지셔닝 전략 (2x2 매트릭스: 기능 복잡도 vs 가격)

4.3 사용자 페르소나 생성

페르소나는 타겟 사용자를 대표하는 가상의 인물입니다. 팀이 동일한 사용자 이미지를 공유하는 데 도움이 됩니다.

페르소나 생성 프롬프트
다음 제품의 사용자 페르소나 3명을 만들어주세요.

제품: 팀 프로젝트 관리 SaaS
대상: 한국 중소기업 (10~50명 규모)

각 페르소나에 포함할 항목:
1. 이름, 나이, 직급, 회사 규모
2. 하루 일과 (간략)
3. 현재 사용 도구
4. 핵심 불편사항 (Pain Points) 3가지
5. 우리 제품에 기대하는 것 (Goals) 3가지
6. 기술 숙련도 (상/중/하)
7. 대표 인용구 ("...")

페르소나 유형:
- Primary: 핵심 타겟 (팀장급)
- Secondary: 보조 타겟 (팀원)
- Negative: 비타겟 (개발자 — 이미 Jira 사용)

좋은 페르소나의 조건

  • 실제 사용자 인터뷰/데이터에 기반할 것 (AI는 초안용)
  • 구체적인 행동과 동기가 포함될 것
  • 팀 전체가 공감할 수 있을 것
  • 의사결정에 실제로 활용될 것
주의: AI가 생성한 페르소나는 가설입니다. 실제 사용자 인터뷰로 검증해야 합니다.

4.4 데이터 수집 및 정리

AI를 활용하면 다양한 형태의 데이터를 구조화하고 분석하는 데 유용합니다.

AI 활용 데이터 정리 시나리오

시나리오 입력 AI 결과물
고객 피드백 분류 피드백 텍스트 목록 카테고리별 분류 + 감성 분석
설문 결과 분석 설문 응답 데이터 통계 요약 + 인사이트 도출
앱 리뷰 분석 앱스토어 리뷰 텍스트 주요 이슈 Top 10 + 개선 제안
경쟁사 가격 정리 각 경쟁사 가격 페이지 URL 통합 가격 비교 테이블
고객 피드백 분류 프롬프트
다음 고객 피드백을 분류하고 분석해주세요.

[피드백 데이터]
1. "로딩이 너무 느려요. 5초 이상 걸려요"
2. "대시보드가 직관적이어서 좋아요"
3. "모바일에서 사용이 불편합니다"
4. "알림 기능이 있으면 좋겠어요"
5. "가격이 경쟁사 대비 비싸요"
... (실제 데이터 붙여넣기)

분석 결과:
1. 카테고리별 분류 (성능/UX/기능요청/가격/기타)
2. 감성 분석 (긍정/부정/중립 비율)
3. 우선순위 높은 이슈 Top 5
4. 개선 제안 (이슈별 액션 아이템)

테이블 형식으로 정리해주세요.

4.5 리서치 결과 보고서 작성

수집한 데이터와 분석 결과를 체계적인 보고서로 정리하는 것이 PM의 핵심 역량입니다.

리서치 보고서 구조

  1. Executive Summary

    핵심 발견사항을 1페이지로 요약합니다.

  2. 조사 방법론

    어떤 데이터를 어떻게 수집했는지 설명합니다.

  3. 주요 발견사항

    데이터에서 도출한 핵심 인사이트를 제시합니다.

  4. 분석 및 해석

    발견사항이 의미하는 바를 해석합니다.

  5. 권고사항

    분석 결과를 바탕으로 구체적인 액션을 제안합니다.

리서치 보고서 프롬프트
다음 리서치 결과를 경영진 보고서로 정리해주세요.

[리서치 개요]
- 주제: 2026년 SaaS 시장 진입 전략
- 방법: 경쟁사 분석(5개사), 사용자 인터뷰(20명), 설문조사(200명)
- 기간: 2026년 3월

[핵심 데이터]
(수집한 데이터 붙여넣기)

보고서 형식:
1. Executive Summary (반페이지)
2. 주요 발견사항 (Top 5, 데이터 근거 포함)
3. 시장 기회 분석 (테이블)
4. 리스크 요인 (매트릭스)
5. 권고사항 (우선순위별 3가지)
6. 다음 단계 (30/60/90일 액션 플랜)

경영진이 5분 내에 읽을 수 있는 분량으로 작성해주세요.
