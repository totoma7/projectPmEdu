2. AI 활용 문서 작성

AI를 활용하여 PRD, 기획서, 요구사항 정의서 등 PM/PL 핵심 문서를 효율적으로 작성하는 방법을 학습합니다.

2.1 PRD(제품 요구사항 문서) 작성

PRD란? Product Requirements Document의 약자로, 제품이나 기능의 목적, 요구사항, 성공 지표를 정의하는 핵심 문서입니다.

AI로 PRD 작성하기 — 단계별 가이드

  1. Step 1: 배경과 목적 정의

    기능의 배경, 해결할 문제, 대상 사용자를 AI에게 전달합니다.

  2. Step 2: 요구사항 도출

    기능적/비기능적 요구사항을 AI와 함께 브레인스토밍합니다.

  3. Step 3: 성공 지표 설정

    KPI와 측정 방법을 구체적으로 정의합니다.

  4. Step 4: 리스크 분석

    예상 리스크와 대응 방안을 AI에게 요청합니다.

프롬프트 예시

PRD 작성 프롬프트
당신은 B2B SaaS 제품의 시니어 PM입니다.

다음 기능의 PRD를 작성해주세요:

## 배경
- 제품: 프로젝트 관리 도구
- 기능: 팀 대시보드 실시간 현황판
- 문제: 팀장이 팀원들의 작업 현황을 파악하기 어려움
- 대상: 5~20명 규모 팀의 팀장

## PRD 구조
1. 기능 개요 (목적, 배경, 대상 사용자)
2. 사용자 스토리 (3가지 이상)
3. 기능 요구사항 (테이블: ID, 요구사항, 우선순위, 상태)
4. 비기능 요구사항 (성능, 보안, 접근성)
5. 성공 지표 (KPI 3가지, 측정 방법 포함)
6. 타임라인 (4주 기준)
7. 리스크 및 대응 방안

마크다운 형식으로 작성해주세요.

실습 과제

여러분의 실제 프로젝트에서 하나의 기능을 선택하여 위 프롬프트 템플릿을 수정하고, AI로 PRD를 생성해보세요.

2.2 기획서/제안서 자동 생성

기획서와 제안서는 이해관계자를 설득하기 위한 문서입니다. AI를 활용하면 초안 작성 시간을 크게 단축할 수 있습니다.

기획서 구조 템플릿

섹션 내용 AI 활용 방법
1. 요약 핵심 내용을 1페이지로 정리 전체 기획 내용을 요약 요청
2. 문제 정의 해결할 문제와 현재 상황 문제 분석 및 근거 데이터 요청
3. 솔루션 제안하는 해결 방안 여러 대안 비교표 생성
4. 실행 계획 일정, 리소스, 마일스톤 WBS 및 간트차트 구조 요청
5. 기대 효과 정량적/정성적 효과 ROI 계산 프레임워크 요청
기획서 작성 프롬프트
다음 프로젝트의 기획서를 작성해주세요.

[프로젝트 개요]
- 프로젝트명: 고객 피드백 자동 분류 시스템
- 목표: CS팀의 피드백 분류 시간을 50% 단축
- 예산: 3,000만원
- 기간: 3개월

포함할 항목:
1. Executive Summary (1페이지)
2. 현재 문제점 분석 (데이터 기반)
3. 제안 솔루션 3가지 비교표
4. 선택한 솔루션의 상세 실행 계획
5. 예상 ROI 및 기대 효과
6. 리스크 매트릭스 (영향도 x 발생확률)

2.3 요구사항 정의서 작성

요구사항 정의서는 개발팀에게 "무엇을 만들어야 하는지" 전달하는 문서입니다.

요구사항 유형

유형 설명 예시
기능 요구사항 (FR) 시스템이 수행해야 할 기능 "사용자는 이메일로 로그인할 수 있다"
비기능 요구사항 (NFR) 성능, 보안, 사용성 등 "페이지 로딩은 3초 이내"
비즈니스 규칙 (BR) 업무 로직과 제약 조건 "무료 플랜은 프로젝트 3개까지"
요구사항 도출 프롬프트
다음 기능의 요구사항 정의서를 작성해주세요.

기능: 팀 멤버 초대 및 권한 관리
대상: SaaS 프로젝트 관리 도구

다음 형식의 테이블로 정리해주세요:

| ID | 유형 | 요구사항 | 우선순위 | 수용 기준 |
|----|------|---------|---------|----------|

- 기능 요구사항 최소 10개
- 비기능 요구사항 최소 5개
- 각 요구사항에 테스트 가능한 수용 기준 포함
- 우선순위: Must/Should/Could/Won't (MoSCoW)

2.4 회의록/보고서 자동 정리

AI를 활용하면 회의록 작성과 보고서 정리 시간을 크게 줄일 수 있습니다.

회의록 정리 워크플로우

  1. 회의 내용 입력

    회의 중 메모한 내용이나 녹취록을 AI에 입력합니다.

  2. 구조화 요청

    AI에게 정해진 형식으로 정리해달라고 요청합니다.

  3. 액션 아이템 추출

    담당자, 기한이 포함된 액션 아이템을 별도 추출합니다.

회의록 정리 프롬프트

[회의 메모]
(여기에 회의 메모를 붙여넣기)

정리 형식:
1. 회의 개요 (일시, 참석자, 목적)
2. 주요 논의 사항 (bullet point)
3. 결정 사항 (번호 매기기)
4. 액션 아이템 테이블:
   | 번호 | 항목 | 담당자 | 기한 | 상태 |
5. 다음 회의 일정 및 안건

    변환된 텍스트를 Claude나 ChatGPT에 입력하여 회의록 형식으로 정리합니다.

  4. 액션 아이템 추출 및 공유

    AI가 추출한 결정 사항과 액션 아이템을 Jira, Notion 등 협업 도구에 등록합니다.

2.5 프롬프트 템플릿 활용법

자주 사용하는 프롬프트를 템플릿화하면 일관된 품질의 문서를 빠르게 생성할 수 있습니다.

PM/PL용 프롬프트 템플릿 라이브러리

문서 유형 핵심 프롬프트 구조 사용 시점
PRD 역할 + 배경 + 기능 개요 + PRD 구조 + 형식 신규 기능 기획 시
사용자 스토리 페르소나 + 기능 설명 + "As a... I want... So that..." 형식 스프린트 계획 시
경쟁 분석 자사 정보 + 비교 항목 + 테이블 형식 + SWOT 전략 수립 시
릴리스 노트 변경사항 목록 + 대상 독자(사용자) + 톤앤매너 배포 직전
주간 보고서 이번 주 성과 + 다음 주 계획 + 이슈/블로커 + 형식 매주 금요일
팁: 프롬프트 템플릿을 노션이나 메모앱에 저장해두고, 상황에 맞게 [변수] 부분만 교체하여 사용하면 효율적입니다.

실습 과제

위 5가지 문서 유형 중 2개를 선택하여 여러분만의 프롬프트 템플릿을 만들어보세요. 실제 프로젝트 맥락을 반영하면 더욱 좋습니다.

2.6 PL을 위한 프로젝트 관리 문서

PL 관점: PM이 "무엇을 만들 것인가"를 정의하는 문서를 작성한다면, PL은 "어떻게, 언제까지 만들 것인가"를 관리하는 문서를 작성합니다.

PL 핵심 문서 유형

문서 목적 AI 활용 방법
WBS (작업 분해 구조) 프로젝트를 관리 가능한 작업 단위로 분해 기능 목록 입력 → AI가 하위 작업 분해
프로젝트 일정표 마일스톤과 작업별 일정 관리 WBS + 리소스 → AI가 간트차트 구조 생성
리스크 관리대장 프로젝트 리스크 식별 및 대응 계획 프로젝트 맥락 → AI가 리스크 도출 및 대응방안 제시
주간 상태 보고서 진척 현황, 이슈, 다음 주 계획 공유 메모/데이터 입력 → AI가 구조화된 보고서 작성
이슈 로그 발생한 이슈 추적 및 해결 현황 관리 이슈 목록 → AI가 영향도 분석 및 우선순위 제안
WBS 생성 프롬프트
당신은 10년 경력의 프로젝트 리더(PL)입니다.

다음 프로젝트의 WBS(Work Breakdown Structure)를 작성해주세요.

## 프로젝트 개요
- 프로젝트명: 고객 피드백 자동 분류 시스템
- 기간: 12주
- 팀 규모: 개발 3명, QA 1명, 디자이너 1명

## WBS 작성 조건
1. 3단계 깊이로 분해 (Phase → Task → Sub-task)
2. 각 작업에 예상 소요일 포함
3. 선행 작업(dependency) 관계 표시
4. 마일스톤 3~4개 지정
5. 테이블 형식으로 정리:
   | WBS ID | Phase | Task | Sub-task | 담당 | 소요일 | 선행작업 |
